# VARAM Website

**Design. Build. Manage.**

Production-ready static multi-page website for VARAM — the built-environment ecosystem uniting Varam Architects, Shri Associates, and MPMS.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Markup | HTML5, semantic elements |
| Utilities | Tailwind CSS v3 (CDN for dev, CLI for production) |
| Custom styles | Per-page CSS files in `css/` |
| JavaScript | Vanilla JS — no framework, no jQuery |
| Icons | Lucide (thin-line, via CDN) |
| Fonts | Jost — Google Fonts (300, 400, 500, 600, 700) |

---

## Setup

### Development (no build step needed)

Open any `.html` file directly in a browser. Tailwind is loaded from CDN with the brand config inlined — no installation required for development.

### Production build

```bash
npm install
npm run build
```

This compiles `src/input.css` → `dist/css/tailwind.css` (minified).

After building, each HTML page's `<head>` already links to `dist/css/tailwind.css`. Remove the CDN `<script>` tags before deploying:

```html
<!-- Remove these two blocks from every page before going live: -->
<script src="https://cdn.tailwindcss.com"></script>
<script>tailwind.config = { … }</script>
```

### Watch mode (hot reload equivalent)

```bash
npm run dev
```

## Deployment

The site is fully static — it can be hosted on:

- **Netlify** — drag-and-drop the `varam-website/` folder, or connect a Git repo
- **Vercel** — `vercel deploy` from the project root
- **GitHub Pages** — push to a `gh-pages` branch
- **Traditional hosting** — FTP the entire folder to any web host

No server, no database, no build pipeline required at the hosting level (build locally, deploy the output).

## Contact Form Setup (Google Apps Script)

1. Go to [script.google.com](https://script.google.com) and create a new project.
2. Paste in a script that:
   - Reads `POST` body (JSON)
   - Sends an email via `MailApp.sendEmail()`
   - Appends a row to a Google Sheet via `SpreadsheetApp`
3. Deploy as a **Web App** — execute as yourself, access by anyone.
4. Copy the deployment URL and replace `[GOOGLE_APPS_SCRIPT_WEBHOOK_URL]` in `js/contact.js`.

## Brand Caveat

The current logo (`varam-logo-dark.svg` / `varam-logo-light.svg`) and yellow + charcoal colour palette belong to **Varam Architects**. Before going live, confirm with the client:

1. Does the umbrella **VARAM** entity have its own distinct logo?
2. Do **Shri Associates** and **MPMS** have their own logos and possibly their own accent colours?
3. Is there a brand guidelines document (exact hex codes, clear-space rules, typography specification)?

The CSS architecture uses per-page stylesheets and CSS custom properties that allow quick introduction of per-vertical accent colour variables without touching shared code.

## Accessibility

The site targets WCAG 2.1 AA compliance:

- Skip-to-content link on every page
- Visible focus states on all interactive elements (`:focus-visible`)
- `aria-label` on all icon-only buttons and links
- `role="dialog"` and `aria-modal="true"` on the mobile drawer and portfolio lightbox
- Semantic HTML5 (`header`, `nav`, `main`, `section`, `article`, `footer`)
- One `<h1>` per page
- `alt` text on every `<img>`
- Sufficient colour contrast (yellow `#EBBE41` on charcoal `#3C3B39` passes AA for large text; verify body-text combinations with a contrast checker before launch)
